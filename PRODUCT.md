# Product

## Register

product

## Users

抓虾服务于电商运营、商品运营、视觉与自动化维护人员。用户通常在桌面端连接已经登录的平台浏览器，按表格或任务模板发起批量自动化任务，并在任务运行后查看产物、审批结果或数据回传。

## Product Purpose

抓虾是一个 Electron + Python 的网页自动化桌面底座，用适配包把森马云盘、天猫、各平台后台和本地数据文件串成可复用任务。当前云端审批方向的目标是把本地必须执行的找图、生图、上传、抓取能力与多人可见的云端审核和数据看板连接起来，让团队共享同一份审批状态和结果数据。

## Brand Personality

稳、准、工作流优先。界面要像可信赖的运营工具，而不是营销型 SaaS 页面；强调任务状态、数据密度、可追溯和明确操作反馈。

## Anti-references

不要把云端审批做成嵌在抓虾里的另一套独立后台，不要重复侧边栏，不要使用营销页式大标题、装饰卡片网格、过度渐变或大面积蓝紫科技感。嵌入抓虾时应保持左侧菜单和右侧工作区的一体化，不出现明显 iframe 边框感。

## Design Principles

1. 本地能力和云端协作分工清楚：本地负责需要登录态和网盘/天猫页面环境的执行，云端负责共享状态、审批、分配和数据沉淀。
2. 任务状态优先于装饰：每个批次、款式、图片、任务机和抓取结果都要有可见状态、下一步动作和失败原因。
3. 复用抓虾桌面壳层：嵌入态使用抓虾现有深色底、橙色主操作、紧凑表格和顶部 tab，而不是另起一套后台导航。
4. 审批动作必须可追溯：AI 图、原始 Prompt、模板版本、重跑记录、确认/舍弃决策和提交任务都要能回查。
5. 多人协作默认安全：账号由管理员创建，任务机通过注册 token 和能力范围授权，云端任务下发必须可审计、可取消、可重试。

## Accessibility & Inclusion

默认按 WCAG AA 处理文字对比和键盘焦点；使用颜色表达状态时必须配合文字标签。产品 UI 动效只用于状态反馈，提供 reduced motion 兼容；中文长字段、长 URL、长 Prompt 和大表格都要避免溢出。
