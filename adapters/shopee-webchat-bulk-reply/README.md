# Shopee 聊聊批量发信

这个适配器会按 Excel 每一行执行下面的动作：

1. 在左侧搜索框搜索买家 ID
2. 读取结果卡片里的站点和店铺信息，匹配目标会话
3. 打开会话后检查是否出现“重新启动对话”
4. 如果出现“重新启动对话”，先点击按钮再继续复制和发送
5. 输入发送话术并点击发送图标
6. 输出结果 Excel，标记成功或失败原因

## Excel 列

- `站点`
- `店铺`
- `买家ID`
- `发送话术`

也兼容英文列名：`site`、`store`、`buyerId`、`message`。

## 运行模式

- `preview`：验证买家搜索、站点/店铺匹配、会话打开和消息输入，不实际发送
- `send`：实际发送消息

## 大表执行

- 支持 `Start Row` / `End Row`，可只跑指定区间，便于断点续跑
- 支持 `Batch Size`，默认每 200 行拆成一个内部批次，按批次队列执行
- 每个批次跑完后会自动翻到下一批，避免整张表在一次 phase 循环里执行到上限
- 超大表建议先用 `preview` 跑一小段，再切到 `send`

## DOM 结论

- 左侧买家搜索框是 `input[placeholder^="搜寻"]`
- 搜索结果列表中的 buyer ID 位于结果卡片正文
- 结果卡片里会显示站点和店铺，例如 `(VN) Balabala Official Store`
- 发送区输入框是底部 `textarea`
- 发送按钮是输入框右下角的 React 图标节点
- 已结束会话会出现 `重新启动对话`，必须先点再发

## 使用建议

- 运行前先登录 [Shopee 聊聊页面](https://seller.shopee.cn/webchat/conversations)
- 如果顶部站点/店铺筛选层正好处于打开状态，建议先刷新页面再运行
- 大批量发送前，先用 `preview` 模式验证 Excel 数据是否匹配
- 长任务可在任务面板里暂停、继续或停止；停止后会保留已完成部分的结果文件
