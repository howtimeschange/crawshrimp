## v1.4.29

### 更新亮点

- 桌面端启动 Python 后端前会先创建运行时根目录和关键子目录，并写入/删除临时探针文件，确认目录真实可写后再启动后端。
- Windows 默认优先使用 `%LOCALAPPDATA%\crawshrimp`，macOS 默认优先使用 `~/Library/Application Support/crawshrimp`，减少用户主目录策略、权限或安全软件导致 `~/.crawshrimp` 不可写时后端无法启动的问题。
- 如果旧 `~/.crawshrimp` 已经存在适配包、adapter-meta、data、knowledge、logs、数据库、配置或 token，会继续优先使用旧目录，避免旧版本用户重新安装后丢失已有运行数据。
- Electron 会把解析后的 `CRAWSHRIMP_DATA` 传给 Python 后端，后端、适配包、导出数据、日志、知识索引、api-token 和 backend.lock 统一落在同一个运行时根目录。
- 每次桌面端启动都会生成独立后端实例 ID，并通过 `/health.runtime` 校验当前连接的后端实例和锁状态，避免误连旧端口上的旧后端进程。
- 同一运行目录下发现旧 backend.lock 或旧后端进程时，桌面端会主动清理并重新拉起后端；macOS Dock 重新打开应用时也会恢复后端服务。
