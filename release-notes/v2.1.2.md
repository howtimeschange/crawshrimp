# 抓虾 Desktop v2.1.2

发布日期：2026-07-13

`v2.1.2` 是桌面自动更新国内下载源与可靠性增强版本：更新发现保持自动，下载与安装仍由用户明确确认。

## Cloudflare R2 更新源

- 正式 `vX.Y.Z` tag 构建完成后，CI 先校验 macOS ZIP、Windows EXE、blockmap 和 `latest*.yml` 的版本与 SHA512。
- 校验通过的更新资产先同步至 `https://updates.crawshrimp.com/`，版本化文件使用长期 immutable 缓存；`latest.yml` 与 `latest-mac.yml` 最后上传并使用重新校验缓存策略。
- R2 元数据读回并与构建产物一致后，才发布 GitHub 版本化 Release 与 `desktop-latest`。GitHub 保留为发布记录、海外手动下载和手动 fallback。
- 从 `v2.1.0` / `v2.1.1` 升级至 `v2.1.2` 的这一跳仍使用 GitHub 更新源；`v2.1.2` 及之后的应用内更新使用 Cloudflare R2。

## 自动检查与提醒

- 应用就绪后 15 秒发起第一次检查，不等待本地 Python 后端完成启动。
- 网络或下载源检查失败后 5 分钟重试；成功后每 6 小时复查。
- 应用重新获得焦点或系统从休眠恢复时，在 5 分钟节流后立即复查。
- 发现新版本显示系统通知；不会自动下载或安装，仍需用户点击侧边栏「更新」。所有检查开始时间都会记录到更新状态。

## macOS 签名稳定性

- macOS 打包只对 Python 运行时中的 Mach-O 可执行文件、动态库和扩展模块执行 Developer ID 签名。
- Python 图片、时区数据库、字节码与其他普通资源不再被误送入 Apple 时间戳签名，避免资源文件触发 `A timestamp was expected but was not found`。

## 验证

- 应用更新单元测试、桌面更新契约测试、适配器 JavaScript 测试与 Python 测试。
- macOS / Windows 正式构建产物、`latest.yml` / `latest-mac.yml` SHA512 校验。
- Cloudflare R2 元数据读回、GitHub 正式 Release 与 `desktop-latest` 资产回读。
