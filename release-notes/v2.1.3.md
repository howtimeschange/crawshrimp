# 抓虾 Desktop v2.1.3

发布日期：2026-07-13

`v2.1.3` 修复了 v2.1.1 等旧版客户端无法跨越到 Cloudflare 更新源的兼容问题，并为 Cloudflare 更新检查加入 GitHub 自动回退。

## 更新源回退

- 应用内更新优先从 `https://updates.crawshrimp.com/` 检查版本与下载资产。
- 如果 Cloudflare 更新检查失败，应用会自动切换到 GitHub 正式 Release 再检查一次；回退成功时仍由用户明确选择下载和安装。
- 每次新的检查周期都会恢复 Cloudflare 优先，避免一次临时网络故障永久改变更新源。

## 旧版升级 Bridge

- `desktop-latest` 重新包含 macOS ZIP、ZIP blockmap、Windows blockmap、`latest-mac.yml` 与 `latest.yml`，与同一 tag 的正式更新资产一致。
- 因此未内置 Cloudflare feed 的 v2.1.1 及更早客户端，仍可从 GitHub 发现并升级到 v2.1.3；这些资产均来自 tag 的 macOS 签名/Apple 公证构建，不使用 main 验证构建。

## 验证

- Cloudflare 首次检查失败、GitHub 回退成功的服务级回归测试。
- `desktop-latest` 完整 bridge 资产集合的工作流契约测试。
- tag 发布后回读 R2 元数据、GitHub 正式 Release、`desktop-latest` 和签名/公证状态。
