## v1.4.30

### 更新亮点

- 桌面端安全链路加固：CI 增加 pull request 构建入口，默认 token 权限收窄，正式 Release 任务按需申请写权限。
- manifest 通知条件不再使用 `eval()` 执行任意 Python 表达式，`/files/delete` 只允许删除运行时数据目录下已登记的任务输出文件。
- 打包下载 Python runtime 时增加 SHA256 校验，避免仅靠 tar 解包校验判断下载可信。
- CDP bridge、浏览器会话下载、通知发送等阻塞路径改为异步友好执行，减少任务运行中阻塞事件循环的风险。
- 任务取消会重新抛出 `CancelledError`，并在二次取消时继续等待部分导出 / 打包收尾，尽量保留已采集数据。
- macOS / Windows 退出流程会检测真实运行中的任务；用户确认后请求停止任务并等待收尾，再关闭后端和受管 Chrome。
- 受管 Chrome 退出前会校验 state file、profile、CDP port 和进程命令行归属，避免误杀用户自己的 Chrome。
- manifest schema 补齐 `line_list` 参数类型，保持 SDK schema 与后端模型一致。
