# 抓虾 Desktop v2.1.0

发布日期：2026-07-11

`v2.1.0` 是抓虾桌面自动更新与安全基线版本。它把版本发现、下载、任务安全检查、重启安装、签名公证、正式 Release 和滚动手动安装包串成一条可验证链路，同时修复本地 Prompt 库、云端审批和 D1 migration 的发布阻断问题。

## 新增：正式桌面自动更新

- 侧边栏底部提供稳定的版本与更新状态：无更新时只显示当前版本，有新版本时才展示检查、下载或安装动作。
- 更新检查、下载与安装均通过受限 IPC 暴露给 renderer；生产包不接受任意更新源覆盖，测试 feed 只允许 loopback HTTP。
- 下载过程展示稳定的百分比与字节进度；磁盘空间不足、元数据异常、SHA 不匹配或网络失败时保留可重试状态。
- 下载完成后不会自动退出或偷偷安装。只有用户点击「重启安装」，且后端确认没有任务阻断后，才进入安装流程。
- Windows 使用 NSIS 在原安装目录就地更新；macOS 使用 ZIP/ShipIt 完成应用内更新，DMG 继续作为首次安装、bridge 覆盖和手动 fallback。

## 加固：任务安全退出与更新安装

- 本地 API 新增原子 update drain：检查普通任务、任务实例、AI 生图任务和云端任务机，并防止检查通过后又启动新任务的竞态。
- 手动任务在创建后台协程前即注册运行 token；`create_task` 失败和后台任务结束都会释放 token，不留下永久阻断。
- AI 生图活跃任务查询不再受 UI 的 500 条列表上限影响，旧的 queued/running 任务也能阻止更新安装。
- 任务状态查询失败、停止后无法确认 drain、受管 Chrome 停止失败时默认 fail-closed；普通退出需要用户显式确认后才会强制继续。
- 安装失败会恢复桌面服务和可重试状态，避免退出流程、后端 drain token 或 updater 状态卡死。

## 改进：本地 Prompt 库优先

- 打开提示词库后先加载并发布本地内容，云端 Prompt 库在后台静默刷新。
- 云端未配置、登录失效或网络失败时不覆盖本地结果，也不会在自动刷新阶段弹出阻断性错误。
- 手动刷新仍保留清晰、友好的云端状态说明，方便需要协作库的用户处理配置或登录问题。
- 补齐 `localPromptLibraryViewState.js` 的 Git 跟踪与契约测试，干净 checkout 不再因本地残留文件掩盖缺失模块。

## 加固：云端审批与 D1 数据完整性

- 任务机只能同步来源机器 ID 与自身精确匹配的现有批次；管理员创建但未指定来源机器的批次不能被任意 active machine 接管。
- 保留管理员显式分配来源机器的能力，同时阻止任务机借已知 `batch_uid` 获得计划素材上传权限。
- 使用 Miniflare 执行真实 D1 migration，覆盖重复任务机指纹归一化、唯一索引约束和 migration 重复执行。
- 补齐 migration 测试 helper 的 Git 跟踪，避免本机测试通过但干净 CI 出现 `TS2307`。

## 更新：运行时与依赖安全基线

- Electron 从 `29.4.6` 升级到 `43.1.0`，同步更新 electron-builder 配置和 CI Node 版本。
- electron-builder 升级到 `26.15.3`，Vite 升级到 `8.1.4`，Vue 插件升级到 `6.0.7`。
- Cloud Excel 解析从 npm 停更的 SheetJS `0.18.5` 切换到官方 `0.20.3` 安全包，Prompt 与测图工作簿导入导出契约保持兼容。
- App 与 Cloud 两棵 npm 依赖树在发布前均达到 `0 vulnerabilities`。

## 发布链路

- `main` push 运行完整测试并构建 macOS/Windows 验证产物，不发布 Release。
- 精确的 `vX.Y.Z` tag 才能发布正式版本，且 tag 必须与 `app/package.json` 完全一致。
- macOS tag 构建要求 Developer ID 签名与 Apple 公证凭据，随后验证 Team ID、Gatekeeper 和 stapler 状态。
- 正式 Release 校验完整更新资产集合后发布；只有版本化 Release 会被标记为 GitHub latest。
- `desktop-latest` 从同一次 tag 构建刷新，只保留三个手动安装包，并将移动 tag 指向相同 release commit。

## 兼容性与升级说明

- 现有 Adapter `manifest.yaml`、JavaScript 注入接口、本地数据目录和 Chrome profile 继续兼容。
- 从 `v2.0.0` 或更早版本升级时，请先从正式 Release 手动下载并覆盖安装 `v2.1.0`；接入后续版本时即可使用应用内更新。
- 源码开发和 CI 使用 Node.js `22.12.0+`；Python CI 继续使用 3.11。
- Cloud 审批台仍是可选模块，本地自动化、AI 生图与本地 Prompt 库不依赖云端配置。

## 发布前验证

- `npm --prefix app test`
- `node --test tests/*.test.js`
- `PYTHONPATH=. python -m unittest discover -s tests -p 'test_*.py' -v`
- `npm --prefix cloud/approval-workbench run check`
- `npm audit`（分别在 `app` 与 `cloud/approval-workbench`）
- unsigned macOS arm64 目录打包，验证 Electron 43、Vite 8、electron-builder 26 与 Python sidecar 组合。
