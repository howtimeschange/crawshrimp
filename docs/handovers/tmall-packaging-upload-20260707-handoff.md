## 本轮代码改动

### 1. PC 详情候选重复序列去重

文件：`adapters/tmall-ops-assistant/tmall-packaging-upload.js`

新增 PC 详情候选的重复序列识别：

- 识别同一批 PC 详情候选中重复的 `01..N` 序列。
- 仅作用于 `pc_detail` 候选，不影响主图、3:4 图、微详情图。
- 必须同时满足：
  - 多个 block 的序号签名一致，例如 `01..11` + `01..11`。
  - 至少一段是模板/通用序列，命中文案如 `模版`、`模板`、`template`、`通用`。
  - 至少一段文件名以当前款号开头，例如 `208126156202_01.jpg`。
- 命中后优先保留款号序列，剔除模板重复序列。
- warning 会写入结果：`PC详情候选检测到 ... 源素材异常，已保留款号序列并剔除模版重复图`。

### 2. `collectPackagingAssets` 带当前款号进入分类器

文件：`adapters/tmall-ops-assistant/tmall-packaging-upload.js`

`collectPackagingAssetsFromSource` 现在调用：

```js
classifyPackagingAssets(assets, { styleCode: job.style_code })
```

这样云盘收集阶段就能知道当前款号，才能在 `126新生儿模版_01..11 + 208126156202_01..11` 这种场景里选择正确的一套。

### 3. 旧版详情 + 手机详情完整链路加强

文件：

- `adapters/tmall-ops-assistant/tmall-packaging-upload.js`
- `core/js_runner.py`
- `tests/tmall-ops-assistant-packaging-upload.test.js`

核心链路已经固化为：

1. 旧版 PC 详情替换后先提交一次 PC。
2. 重新进入编辑页。
3. 打开手机端详情编辑器。
4. 清空旧手机端模块。
5. 导入电脑端详情。
6. 保存。
7. 完成编辑。
8. 回到编辑页后再次最终提交。
9. 最终读回校验 PC 和手机端详情。

`core/js_runner.py` 增加 `cdp_target_eval`，用于在匹配到的 CDP target 内执行表达式，解决手机详情编辑器跨域 iframe 内部 API/React 状态不可直接由主页面访问的问题。

### 4. 新增/加强测试

文件：`tests/tmall-ops-assistant-packaging-upload.test.js`

关键新增覆盖：

- `packaging classifier dedupes PC detail template sequence and keeps style-code sequence`
  - 构造 `126新生儿模版_01..11 + 208126156202_01..11`。
  - 断言只保留 11 张。
  - 断言保留下来的文件名都以 `208126156202_` 开头。
  - 断言 warning 包含 `源素材异常` 和 `保留款号序列`。
- `collectPackagingAssets passes style code when deduping duplicate PC detail sequences`
  - 覆盖云盘 collect 层，不只是分类器 helper。
  - 确认 `collectPackagingAssets` 也能只保留款号序列。
