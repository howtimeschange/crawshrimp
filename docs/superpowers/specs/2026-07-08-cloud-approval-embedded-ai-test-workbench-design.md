# 抓虾云端审批嵌入与 AI 测图工作台设计

## 背景与当前诊断

当前分支已经实现了云端审批工作台的基础能力：云端账号/RBAC、任务机注册鉴权、心跳与 claim、审批批次同步、单图确认/舍弃、舍弃图重生图 job、提交天猫测图 job、以及钉钉通知里的云端审批链接。

这次反馈暴露出两个核心问题：

1. 抓虾客户端里内嵌云端审批页无法登录。云端登录依赖 `cs_session` HttpOnly Cookie，当前 Cookie 是 `SameSite=Lax`；当页面以 `http://127.0.0.1:5173` 的 Electron renderer 内 iframe 加载 `https://approval.crawshrimp.com` 时，云端站点处于第三方 iframe 语境，登录响应即使成功，后续 `/api/auth/me` 也可能不带 Cookie，所以页面看起来像“输入账号密码后无反应”。
2. 嵌入体验不够像抓虾的一部分。抓虾本体已经是左侧菜单 + 右侧工作区，`CloudApprovalFrame.vue` 又额外加了标题、状态 toolbar、iframe 边框，云端 `App.vue` 内部还有独立侧边栏，因此视觉上像在抓虾里扣了一个独立后台。

现有能力与缺口：

- 已有：云端审批批次、图片资产上传、任务机派发、重生图、提交天猫测图、基础 Prompt CRUD、基础数据看板。
- 缺口：嵌入态会话稳定性、无边框一体化 UI、云端 Prompt Excel 导入导出/在线表格、云端在线生图入口、云端图片资源库语义、AI 测图数据抓取入库/看板、一次性/定时抓取任务派发。

## 推荐架构

采用“云端独立网页 + 抓虾嵌入态适配”的双形态，而不是把所有云端能力复制到 Electron 本地页面。

### 独立网页

独立地址继续是 `https://approval.crawshrimp.com`，用于多人在线审核、账号管理、任务机管理、Prompt 库维护和数据看板。它保留完整 RBAC、管理员创建账号、不开放注册。

### 抓虾嵌入态

嵌入态默认仍打开同一套云端工作台，但必须进入 `embed=1` 模式：

- 隐藏云端内部侧边栏。
- 使用顶部 tab 切换：`审批批次`、`AI 生图`、`Prompt 库`、`测图数据`、`任务机`、`账号`。
- 内容区直接铺满抓虾右侧工作区，不展示额外 iframe 边框和云端独立品牌侧栏。
- 颜色、圆角、按钮、表格密度对齐抓虾现有 token：深色背景、橙色主操作、紧凑表格、8px 左右圆角。

登录问题先用 Cookie 策略修复：云端 session Cookie 支持嵌入语境下的 `SameSite=None; Secure`，并为 Electron iframe 加登录回归测试。若 Electron 后续仍受第三方 Cookie 策略影响，再升级为本地 BFF 会话代理：抓虾本地后端保存云端用户 session，renderer 只访问本地 `/cloud-approval/user/*` 和 `/cloud-approval/proxy/*`。

## 业务闭环 1：找图、生图、审批、重跑、提交天猫

1. 抓虾客户端读取 `AI测图任务导入模板.xlsx`。
2. 抓虾在森马云盘按款号找主图/参考图。
3. 抓虾把源图/参考图上传到云端图片资源库，同时创建或更新 AI 测图批次。
4. 业务在云端或本地触发生图：
   - 云端触发时创建 `generate_ai_image` job，由任务机领取执行。
   - 本地触发时沿用当前异步 1XM 生图能力，完成后同步云端。
5. 生成结果进入审批看板，业务可单图确认/舍弃、换 Prompt、批量重跑所有舍弃图或未完成款式。
6. 重跑任务创建 `generate_ai_image` 或 `regenerate_ai_image` job，新图追加到同一款式，不覆盖历史图。
7. 单款至少一张确认图后进入 ready；全部款式 ready 后可选择任务机提交天猫测图。
8. 任务机执行本地天猫上传/创建测图任务，云端回写结果、证据文件和失败原因。

## 业务闭环 2：测图数据抓取与云端看板

1. 云端 AI 测图任务看板展示批次、款式、图片、素材 ID 与数据表现。
2. 用户可点击“立即抓取”或配置定时抓取。
3. 云端创建 `crawl_tmall_material_test_data` job，指定任务机、统计口径、时间范围、商品 ID 范围。
4. 任务机调用现有 `巴拉-AI测图数据抓取导出` 能力，在本地已登录天猫环境里导出数据。
5. 任务机把导出的 Excel 上传云端，并提交解析后的 `概览/明细` 行。
6. 云端入库后刷新看板：按款式、商品、图片类型、Prompt 模板、素材 URL 展示曝光、点击、点击率、详情点击、加购、支付转化等指标。

## UI 方向

这是一套高密度运营工具，而不是营销页面。默认采用 restrained product UI：

- 保留抓虾左侧菜单，只在右侧内容区切换云端工作台。
- 云端内部使用顶部 tab，不再使用左侧 sidebar。
- 首屏优先显示当前待处理批次、任务机在线状态、最近抓取结果和失败事项。
- 表格行内提供主要动作，少用弹窗；只有上传文件、批量操作确认、危险动作撤销需要弹窗。
- 图片审批页以款式为主轴，左侧/顶部可筛选状态，右侧详情区展示 Prompt、参考图、生成历史和重跑参数。

## 权限与安全

- 用户账号仍由管理员创建，不开放注册。
- 嵌入态和独立网页使用同一套 RBAC。
- 任务机只使用 machine token，不能访问用户后台 API。
- 注册 token 只显示一次，保存 hash，可限制能力、有效期、是否需要审批。
- 云端派发任务使用 job 状态机和租约；上传创建任务使用幂等键，避免重复创建天猫测图任务。

## 验收标准

1. 在抓虾 `codex/cloud-approval-workbench` 分支客户端中，云端审批菜单打开后可正常登录。
2. 登录后右侧内容区看起来像抓虾工作区的一部分，无云端独立侧边栏、无明显 iframe 边框。
3. Prompt 库可导入用户 Excel，导入后可在线表格编辑，可导出同字段 Excel。
4. 云端 AI 生图入口能选择款式、主图/参考图、Prompt 模板，并通过任务机异步生成结果。
5. 审批看板能确认/舍弃/批量重跑/换 Prompt，新图追加到列表。
6. 确认后可选择任务机下发天猫测图提交任务，并回传结果。
7. 云端数据看板可导入或通过任务机抓取 `巴拉-AI测图数据抓取导出` 结果，并按真实表字段回显核心指标。
