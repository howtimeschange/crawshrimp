# AI 生图加载态优化设计

## 目标

优化 AI 生图工作台队列卡片的加载反馈，并移除结果区顶部的模型、比例、尺寸、张数和状态摘要。

## 方案选择

考虑过三种实现：

1. 只替换现有 CSS 占位动画。改动最小，但不能体现主图或参考图。
2. 直接读取左侧当前表单的主图。实现简单，但切换任务或修改配置后会让历史队列显示错误图片。
3. 每条加载卡片从所属任务或生成 run 的持久化参数中解析加载预览。能够保持队列隔离，并兼容单次生成和异步批量生成。

采用方案 3。

## 加载卡片规则

- 有主图时，使用该次生成的主图作为卡片背景。
- 无主图但有参考图时，使用第一张参考图作为卡片背景。
- 没有任何图片输入时，展示抓虾主题的默认图片骨架。
- 输入图片背景使用放大、模糊、轻微降饱和处理，并叠加暗色遮罩、呼吸光和进度动画，避免被误认为生成结果。
- 卡片下方展示循环加载文案：
  - 正在出海
  - 正在撒网
  - 正在寻找灵感海域
  - 正在捕捞画面
  - 正在收网
  - 正在挑选大虾
  - 正在满载返航
- 文案按固定时间间隔循环，同一时刻所有加载卡片可使用基于卡片索引的错位文案，避免机械地同步切换。
- queued 和 running 都使用同一视觉组件；状态辅助文字仍保留“排队中”或“生成中”，满足可访问性和状态识别。

## 数据来源

- 持久化异步 run 优先读取 run 自身的输入参数；缺失时回退到所属 job 的 `params.main_image_path` 与 `params.reference_image_paths`。
- 当前单次生成的临时加载卡片读取提交时的表单输入快照。
- 本地路径继续复用现有 `imagePreviewSrc()` / `refreshImagePreview()` 机制，不新增下载接口。
- 预览加载失败时自动回退到默认图片骨架，不影响任务轮询。

## 顶部结果区

- 保留任务标题、选择状态、全选、下载和任务侧栏按钮。
- 删除 `GPT Image 4K · 3:4 · 2448x3264 · 1 张 · completed` 这类摘要行。
- 左侧“下次生成参数”和右侧任务记录中的参数信息保持不变。

## 生命周期

- 组件挂载时启动一个轻量文案轮换计时器。
- 仅在存在加载卡片时更新文案索引。
- 组件卸载时清理计时器，避免后台持续更新。
- 遵守 `prefers-reduced-motion`：关闭呼吸、漂移和闪烁动画，但保留静态背景与状态文字。

## 测试与验收

- 结构测试锁定三类预览选择规则、文案集合、计时器清理和顶部摘要删除。
- Vite 构建必须通过。
- 在 `127.0.0.1:5173` 验证：
  - 有主图的 loading 卡片显示模糊主图。
  - 只有参考图时显示第一张参考图。
  - 纯文字生图显示默认抓虾图片骨架。
  - 文案能够循环，queued/running 状态仍可识别。
  - 顶部摘要不再出现。
