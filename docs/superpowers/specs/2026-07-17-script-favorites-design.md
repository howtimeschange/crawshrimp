# 我的脚本收藏设计

## 目标

在“我的脚本”中让用户收藏常用脚本，并在页面顶部优先看到最近收藏的脚本。收藏在桌面壳和浏览器开发模式中共用，并在应用重启后保留。

## 范围

- 在每张脚本卡片上提供独立的星标按钮。
- 将页面分成“我的收藏”和“全部脚本”两个区域。
- 收藏数据由核心服务持久化。

不包含跨设备同步、收藏分组、批量收藏或搜索筛选。

## 数据与接口

核心服务配置增加非敏感字段：

```json
{
  "script_favorites": {
    "<adapter_id>": "<ISO-8601 UTC timestamp>"
  }
}
```

收藏接口只接受当前已安装的 `adapter_id`：

- `GET /script-favorites` 返回收藏映射。
- `PUT /script-favorites/{adapter_id}` 写入当前 UTC 时间并返回更新后的映射。
- `DELETE /script-favorites/{adapter_id}` 删除该键并返回更新后的映射。

通过 Electron preload 暴露同名的 `window.cs` 方法；浏览器开发桥接走同一组 HTTP 接口。移除脚本时同步清理其收藏记录，避免配置残留。

## 页面交互

1. 列表初次加载时，同时获取脚本和收藏映射；收藏请求失败不阻塞脚本列表，显示可理解的操作错误并保留已加载列表。
2. 每张卡片右上角有可访问的星标按钮。未收藏为描边星，收藏为高亮实心星，并以 `aria-pressed` 表示状态。
3. 点击星标必须阻止卡片的“打开脚本”事件。收藏或取消收藏成功后即时刷新两区；请求进行中禁用该按钮，防止并发反转。
4. “我的收藏”只显示已收藏的卡片，按照 `favorited_at` 倒序；相同或无效时间用稳定的脚本原始顺序兜底。
5. “全部脚本”显示未收藏的卡片，保留当前后端返回的顺序。收藏区为空时不展示空标题或占位块。
6. 重新收藏同一个已取消收藏的脚本会写入新的时间，因此排到收藏区最前。

## 组件边界

- 核心服务负责收藏映射的校验、读写和随卸载清理。
- preload 与开发桥接只负责将调用转发给核心服务。
- `ScriptList.vue` 负责并行加载、乐观以外的成功后更新，以及把原始脚本组派生为两个展示区；原有进度、任务标签、打开卡片和移除行为保持不变。
- 纯排序/分区逻辑提取到 renderer 工具函数，以便无需 DOM 即可测试。

## 错误处理

- 核心服务拒绝未知或空的 `adapter_id`，不写入配置。
- 读到非对象或非法时间戳时忽略该条收藏记录，不影响其他脚本。
- 收藏写入失败时，星标与分区不改变，并显示失败信息；用户可再次点击重试。

## 验证

先写失败测试并确认 RED，再实现：

1. 核心服务测试：收藏写入时间、取消删除、未知脚本拒绝、卸载同步清理及配置重启读回。
2. renderer 工具测试：收藏按时间倒序、未收藏脚本保留原顺序、非法时间的稳定兜底。
3. UI 接线测试：星标使用独立点击事件，收藏和全部区域绑定正确的派生集合，并且浏览器开发桥接具备收藏接口。
4. 运行相关 Node/Python 测试与前端构建；在 Electron 壳手动确认收藏、取消、重启后保留和最近收藏前置。
