# AI 生图任务置顶与删除设计

## 目标

为 AI 生图工作台的任务记录增加持久化置顶、取消置顶和安全删除能力。删除任务时同步清理抓虾运行目录中的任务图片缓存，但不得删除用户通过“下载”或“另存”保存到本地文件夹的副本。

## 用户交互

任务记录中的每个任务改为独立卡片：

- 卡片主体用于切换任务，保持当前任务的高亮边框和缩略图、最近生成时间、结果数量信息。
- 卡片右上角显示“置顶”或“取消置顶”。点击后立即更新排序；后置顶的任务排在先置顶任务之前。
- 卡片右下角显示“删除”。排队中或生成中的任务不允许删除，按钮禁用并提示“任务生成中，完成或失败后可删除”。
- 点击可删除任务的“删除”后打开二次确认弹窗，不直接执行删除。

确认弹窗文案为：

> 删除任务“{任务名称}”？删除后将清除任务记录和未下载的本地图片缓存。已下载或另存到本地文件夹的图片不受影响。此操作无法撤销。

弹窗提供“取消”和“确认删除”。确认按钮使用危险操作样式，提交期间禁用两个按钮并显示“删除中…”。

## 持久化与排序

`ai_image_jobs` 新增 `pinned_at` 文本字段，默认空字符串：

- 置顶时由后端写入当前 UTC 时间，不允许前端指定时间。
- 取消置顶时清空 `pinned_at`。
- 置顶操作不修改 `updated_at`，避免取消置顶后改变普通任务的原始生成时间顺序。
- 列表排序依次为：已置顶优先、`pinned_at` 倒序、`updated_at` 倒序、`id` 倒序。

后端新增 `PATCH /ai-image/jobs/{job_uid}/pin`，请求体为 `{ "pinned": true | false }`。Electron 主进程、preload 和开发浏览器桥接统一暴露 `setAiImageJobPinned(jobUid, pinned)`。

## 删除与缓存边界

后端新增 `DELETE /ai-image/jobs/{job_uid}`，由 AI 图片服务协调缓存清理和数据删除：

1. 读取任务及其素材、画布数据；任务不存在返回 404。
2. 若任务状态或任一运行记录为 `queued`、`running`，返回 409，不删除任何内容，也不取消 1XM 任务。
3. 从任务、运行摘要、素材和画布中递归收集路径，只接受解析后位于 `runtime_paths.child_dir("ai-image-cache")` 内的文件。
4. 补充匹配名称中包含该任务 UID 前八位的任务缓存文件，覆盖未成功写回数据库的标注图或临时物化结果。
5. 依次删除缓存文件。文件不存在视为成功；若有文件因权限或占用删除失败，返回错误并保留任务记录，用户可重试。
6. 缓存清理成功后，在同一数据库事务中删除该任务的 `ai_image_assets`、`ai_image_canvases` 和 `ai_image_jobs` 记录。

任何不在 `ai-image-cache` 下的路径一律不得删除，包括：

- 任务的 `output_dir`；
- 默认的 `~/Downloads/抓虾导出/AI生图`；
- 用户通过下载或另存选择的任意目录；
- 作为主图或参考图上传的原始文件。

删除接口返回删除的缓存文件数量，供前端提示“任务已删除，已清理 N 个缓存文件”。

## 前端状态处理

- 置顶成功后以服务端返回任务更新本地 `jobs`，随后重新加载列表，保证排序与数据库一致。
- 删除成功后清理该任务对应的 `taskDrafts`、缩略图预览、结果缓存映射、选中结果、修图会话结果和轮询状态。
- 删除当前任务时，从刷新后的排序列表中打开第一条剩余任务；没有剩余任务时恢复为空白的新任务表单，但不立即创建数据库记录。
- 删除非当前任务不得打断当前任务，也不得改变当前表单。
- 后端返回 409 时关闭提交态并显示运行中任务不可删除的提示；其他失败保留弹窗，允许重试或取消。

## 可访问性

- 任务卡不使用嵌套按钮。卡片采用容器元素，内部使用一个任务选择按钮和两个独立操作按钮。
- 删除确认框使用 `role="dialog"`、`aria-modal="true"` 和清晰的可访问名称。
- 弹窗打开后聚焦第一个可用按钮，Tab 焦点限制在弹窗内；Escape 在未提交时关闭并将焦点还给触发删除的按钮。
- 置顶、取消置顶和删除按钮都有独立可访问名称，禁用删除按钮提供原因说明。

## 错误处理

- 任务不存在：前端从本地列表移除失效记录并刷新任务列表。
- 运行中任务删除：后端返回 409，前端展示业务提示，不记录为未知错误。
- 缓存清理失败：任务记录保持存在，弹窗展示失败信息，用户可重试。
- 置顶更新失败：保持现有顺序并展示错误提示。

## 测试与验收

自动化测试覆盖：

- 数据库字段迁移、置顶顺序、后置顶优先、取消置顶恢复普通排序；
- 删除任务级联删除素材和画布；
- 运行中任务返回 409 且数据、缓存均保留；
- 仅删除 `ai-image-cache` 内的任务文件，用户下载目录文件保持不变；
- 缓存删除失败时任务记录保持存在；
- API 路由、Electron IPC、preload 和开发桥接契约；
- 任务卡无嵌套按钮，置顶/删除位置和文案、确认弹窗、焦点与 Escape 行为；
- 删除当前任务后的任务切换和空列表状态。

真实界面验收使用临时创建的未生成任务，不触发 1XM：依次验证后置顶任务排在前、取消置顶恢复、删除弹窗文案、取消不删除、确认后任务消失。另用一个测试缓存文件验证删除后缓存消失、外部下载副本仍存在。
