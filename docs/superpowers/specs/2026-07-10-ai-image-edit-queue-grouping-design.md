# AI 生图修改结果同队列展示设计

## 目标

AI 生图工作台不再把每次修改图生成显示为新的独立队列。修改结果应回到其来源图片所在的原始队列：原图结果在前，后续修改结果按生成记录的创建顺序依次追加，从而在结果墙中直接表达同一段 Prompt 的持续修改关系。

本次只调整工作台展示投影。后端继续将每次生成保存为独立 run，并继续使用 `edit_source.result_key` 记录父图，不改变 1XM 异步任务、轮询、重试或 AI 测图脚本接口。

## 队列归并规则

1. 没有 `edit_source` 的 run 是根队列。
2. 修改 run 通过 `edit_source.result_key` 找到父图，再沿父链找到根图及其根队列。
3. 根 run 的全部结果保持在前；归属于该根队列的修改 run 按任务摘要中的 run 创建顺序追加。
4. 合并后图片在队列内统一重新编号为“结果 1、结果 2……”；不再出现多个独立的“结果 1”分组。
5. 排队中、生成中或失败的修改 run 同样追加到来源队列末尾，保证加载态和重试入口出现在正确关系中。
6. 如果历史数据缺少父图、`result_key` 无法解析或形成异常循环，该 run 保留为独立队列，不能隐藏结果。
7. 多 Prompt 批量生成的各根 run 仍是独立队列，不允许因同属一个任务而互相混合。

## 分支与大图模式

结果墙展示根队列下所有修改结果的生成顺序，包括从同一原图产生的兄弟分支。

大图模式继续按当前父子分支展示：图 1 → 图 2 → 图 3 时查看图 3 显示 1、2、3；从图 1 另开分支生成图 4 时查看图 4 只显示 1、4。队列归并不改变大图的分支筛选和 Prompt 修改链。

## 队列元数据

- 标题和 Prompt 使用根 run 的信息。
- 图片数量使用归并后的实际卡片数量。
- 状态按子 run 聚合：生成中优先于排队中，排队中优先于失败，失败优先于完成。
- 生成时间保留根 run 的生成时间，避免每次修改导致队列身份和排序抖动。

## 测试与验收

自动化测试覆盖：

- 原图、连续两次修改归并为一个队列并按顺序编号；
- 从原图产生兄弟分支时仍归并到同一结果队列；
- 多张原始结果位于修改结果之前；
- 进行中和失败的修改占位卡进入来源队列；
- 多 Prompt 根队列保持分离；
- 父图缺失时修改 run 保留独立队列；
- 大图父子分支与 Prompt 修改链行为不回退。

真实页面使用已授权的 1XM 请求验证：创建一个独立 QA 任务，完成文字生图、连续修改、从原图分支修改，并在切换任务和刷新页面后确认仍只显示一个根队列、卡片顺序稳定、大图分支正确。测试任务保留供用户复核，不删除其生成素材。
