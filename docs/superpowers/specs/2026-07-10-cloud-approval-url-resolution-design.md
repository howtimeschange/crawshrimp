# 云端审批地址自动解析设计

## 背景

抓虾当前把 `cloud_approval.base_url` 作为可编辑设置暴露给业务用户。用户需要理解本地开发端口、正式环境域名以及地址格式，历史配置还可能让 Prompt 库、任务机和审批页面继续连接到已经失效的服务。

本改动采用方案 A：运行环境决定云端审批地址，设置页只展示后端解析出的有效地址，不再让业务用户输入。正式安装包固定连接正式域名；本地开发由后端发现正在运行的审批服务，并以 `8787` 作为默认端口。

## 目标

1. 正式安装包始终使用 `https://approval.crawshrimp.com/`。
2. 本地开发允许审批服务使用变化的端口，并由抓虾后端自动发现。
3. Prompt 库、任务机注册与轮询、审批页面和批次同步使用同一个有效地址。
4. 设置页地址只读，并明确展示地址来源和本地服务是否已被检测到。
5. 保留已有任务机凭据、注册流程和其他云端审批设置的兼容性。

## 非目标

1. 不改变云端审批 Worker 的部署方式或线上域名。
2. 不自动启动或停止本地云端审批服务。
3. 不扫描局域网、非回环地址或任意大范围端口。
4. 不迁移或删除历史配置中的 `cloud_approval.base_url`；它只是不再作为正式环境的有效地址。
5. 不调整 Prompt 库、任务调度、租约或任务机鉴权协议。

## 方案比较

### 方案 A：后端集中解析（采用）

Electron 启动 Python 后端时注入运行环境，Python 后端集中完成正式地址固定、本地健康检查、端口探测和回退。所有消费者只读取后端解析结果。

优点是地址规则只有一份，直接启动 Python 的开发与测试场景也可覆盖；缺点是后端需要增加轻量健康探测和短时缓存。

### 方案 B：Electron 探测后传入地址

Electron 在启动 Python 前探测端口，并把最终地址作为环境变量传入。实现入口直观，但 Electron 和 Python 会分别持有地址规则，直接运行 Python 时缺少同等行为，后续容易产生两套逻辑。

### 方案 C：审批服务写入端口发现文件

本地审批服务启动时把地址写入共享运行目录，抓虾后端读取该文件。它能准确获取随机端口，但会耦合两个独立服务的启动和清理流程，异常退出后还需处理陈旧文件。

## 环境判定

Electron 已能通过 `app.isPackaged` 判断是否为安装包。启动 Python 后端时，它注入：

- `CRAWSHRIMP_APP_ENV=production`：正式安装包。
- `CRAWSHRIMP_APP_ENV=development`：Electron 开发模式。

直接运行 Python 后端且未设置该变量时按 `development` 处理，保持现有本地开发命令可用。打包入口必须显式注入 `production`，相关行为由 Electron 主进程测试锁定。

## 地址解析规则

后端新增单一有效地址解析器。所有云端审批客户端和状态接口都通过它取得地址，不再直接读取 `cloud_approval.base_url`。

### 正式环境

1. 有效地址固定为 `https://approval.crawshrimp.com/`。
2. 忽略 `CRAWSHRIMP_CLOUD_APPROVAL_URL` 和历史保存地址，防止安装包被旧配置或普通环境变量指向其他服务。
3. 状态中的地址来源为 `production_default`。

### 本地开发环境

按以下优先级解析：

1. `CRAWSHRIMP_CLOUD_APPROVAL_URL`：开发者显式覆盖。该值是权威选择，不因探测失败而静默切换到其他端口；后端仍检查健康状态并向界面报告错误。
2. 历史保存的回环地址：仅当主机为 `127.0.0.1`、`localhost` 或 `::1`，且健康检查通过时复用。非回环历史地址不参与自动发现。
3. 受限端口探测：检查 `http://127.0.0.1:8787` 至 `http://127.0.0.1:8797`，选择第一个通过身份检查的地址。
4. 默认回退：没有发现服务时返回 `http://127.0.0.1:8787`，但标记为未检测到服务。

地址统一移除末尾 `/` 后用于 API 拼接；界面可按标准格式展示。显式覆盖仅接受完整的 `http://` 或 `https://` URL。若缺少协议或 URL 无效，解析结果保留来源 `environment_override`，但返回空的有效地址、`configured=false` 和 `invalid_environment_override` 错误，不探测或回退到其他地址。

## 服务身份检查

自动发现不能只判断端口是否可连接。候选地址必须满足：

1. `GET /health` 在短超时内返回成功。
2. 响应是 JSON。
3. `ok === true`。
4. `service === "crawshrimp-cloud-approval-workbench"`。

端口被其他本地服务占用、返回非 JSON、超时或服务标识不匹配时，候选地址无效。探测只访问回环地址，并使用有限端口范围和短超时，避免拖慢设置页和后端启动。

自动发现结果缓存 10 秒，正常消费者读取缓存，避免重复扫描。`GET /cloud-approval/status?refresh=true` 绕过缓存并重新探测；设置页每次进入云端审批面板时使用该参数。连接失败会立即让对应缓存失效，使下一次状态刷新可以发现新端口。并发刷新由后端锁合并为一次探测，所有等待者获得同一份结果。

## 后端状态契约

`GET /cloud-approval/status` 保留现有字段，并增加：

- `base_url`：解析后的有效地址。
- `base_url_source`：`production_default`、`environment_override`、`saved_local`、`detected_local` 或 `development_fallback`。
- `service_reachable`：是否通过审批服务身份检查。
- `service_error`：未通过时的稳定错误码；成功时为空字符串。首版错误码为 `invalid_environment_override`、`unreachable`、`unexpected_service` 和 `not_detected`，由界面映射为中文提示。
- `environment`：`production` 或 `development`。

现有 `configured` 表示系统已有可使用的地址规则。正式环境和开发回退都为 `true`；无效的开发环境显式覆盖为 `false`。界面使用 `service_reachable` 区分“已有地址”和“当前检测到服务”。

`POST /cloud-approval/config` 暂时保留 `base_url` 请求字段以兼容已有调用方，但不再把请求值作为有效地址。保存任务机名称、注册 token、启用状态和能力时，不应覆盖后端解析结果。

## 消费者统一

以下入口必须使用有效地址解析器：

1. 任务机注册、启动、心跳、领取任务和结果回传。
2. Prompt 库列表、模板解析、导入和同步。
3. 本地批次同步到云端审批。
4. Electron 内嵌审批页面和审批链接可信来源校验。
5. 云端审批状态展示。

解析器是唯一地址来源，消费者不得自行回退到配置文件地址。

## 设置页交互

1. “云端地址”字段保持可复制，但设置为只读，移除要求用户输入的占位提示。
2. 正式环境显示固定线上地址，并提示“正式环境固定地址”。
3. 开发环境探测成功时显示实际地址，并提示“已检测到本地审批服务”。
4. 开发环境回退时显示 `http://127.0.0.1:8787`，并提示“未检测到本地审批服务，当前显示默认地址”。
5. 显式覆盖无效、服务不可达或身份不匹配时显示后端返回的稳定错误信息。
6. “保存配置”和“注册任务机”继续保存或提交任务机相关字段，但不提交用户可编辑的地址。

本改动不增加端口输入框。开发者需要指定扫描范围以外的地址时，通过 `CRAWSHRIMP_CLOUD_APPROVAL_URL` 配置。

## 错误处理

1. 正式地址暂时不可达时仍保留固定地址，不回退到本地或历史地址。
2. 开发显式覆盖不可达时保留覆盖地址并报告错误，不静默连接其他本地服务。
3. 自动探测遇到错误时继续检查下一个候选端口；所有候选均失败后使用默认回退。
4. 依赖云端的操作在 `service_reachable=false` 时给出明确错误，不能只提示“未配置云端地址”。
5. 地址变化不清理任务机凭据；若新服务拒绝现有机器凭据，沿用已有 `machine_token_invalid` 恢复流程。

## 测试设计

### Python 后端

1. 正式环境始终返回固定线上地址，并忽略环境覆盖和历史地址。
2. 开发环境显式覆盖优先，且不可达时不会切换到扫描结果。
3. 可用的历史回环地址优先于端口扫描，非回环历史地址被忽略。
4. 探测只接受正确的 `/health` 服务标识，并限制在 `8787–8797`。
5. 未发现服务时返回 `8787` 回退地址和 `service_reachable=false`。
6. 所有云端客户端构造都使用解析后的地址。
7. 配置保存不再改变有效地址，但仍保存其他任务机字段。
8. 缓存命中、失效和并发读取不会返回互相矛盾的地址与状态。
9. `refresh=true` 绕过 10 秒缓存并合并并发探测。

### Electron 与渲染层

1. Electron 开发和打包模式向后端注入正确的 `CRAWSHRIMP_APP_ENV`。
2. 设置页地址输入框为只读。
3. 设置页根据 `base_url_source`、`service_reachable` 和 `service_error` 展示正确提示。
4. 保存配置和注册任务机不会把界面地址作为可编辑配置提交。
5. Prompt 库、审批页面和任务机 IPC 继续从后端状态取得同一地址。

### 回归验证

1. 运行云端审批、设置页、IPC、Prompt 库和任务机相关测试。
2. 本地分别在 `8787` 和扫描范围内的另一个端口启动审批服务，确认地址能自动变化。
3. 在 `8787` 启动一个非审批 HTTP 服务，确认不会误识别。
4. 模拟正式环境，确认历史本地地址无法覆盖线上域名。

## 兼容与迁移

不删除用户现有 `config.json` 字段，也不迁移任务机凭据。升级后：

- 正式安装包立即使用固定线上地址。
- 开发环境只复用通过身份检查的历史回环地址。
- 其他历史地址被保留但不参与有效地址解析。
- 回滚旧版本时，旧配置仍可读取，不会因为本改动丢失。

## 验收标准

1. 新安装正式客户端无需填写地址即可看到 `https://approval.crawshrimp.com/` 并注册任务机。
2. 本地审批服务运行在 `8787–8797` 任一端口时，抓虾后端能识别并让设置页只读显示该地址。
3. 未启动本地审批服务时，设置页显示默认 `8787` 地址和明确的未检测提示。
4. 任意业务界面都不能修改有效地址。
5. Prompt 库、任务机和审批页面在同一时刻使用相同地址。
6. 现有任务机名称、能力、启用状态和凭据不因升级丢失。
