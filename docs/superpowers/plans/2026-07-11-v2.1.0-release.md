# Crawshrimp v2.1.0 Release Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Publish a verified Crawshrimp Desktop `v2.1.0` release from the current `main`, including complete documentation, GitHub CI artifacts, the formal versioned release, and the rolling `desktop-latest` release.

**Architecture:** Keep `app/package.json` as the canonical desktop version and require the tag to match it exactly. Ship the existing desktop lifecycle, local-first Prompt library, cloud approval, migration, and Electron security fixes as one release commit, then let `.github/workflows/build-desktop.yml` build, sign, notarize, validate, and publish artifacts.

**Tech Stack:** Electron 43, Vue 3, Node.js 22.12, Python 3.11, Cloudflare Workers/D1/R2, GitHub Actions, electron-builder.

## Global Constraints

- Release version is exactly `2.1.0`; Git tag is exactly `v2.1.0`.
- Preserve all untracked runtime trees such as `.crawshrimp-dev*`; never stage them.
- Do not push until local app, root Node, Python, and cloud checks pass.
- A release is complete only after both `main` and `v2.1.0` workflows succeed, the formal release has the expected updater assets, and `desktop-latest` points to the release commit.

### Task 1: Release Scope and Blocker Audit

**Files:**
- Review: all tracked working-tree changes
- Review: `.github/workflows/build-desktop.yml`

**Interfaces:**
- Consumes: current `main`, current tracked diff, GitHub remote state
- Produces: an explicit go/no-go decision and a bounded release file list

- [ ] **Step 1: Inspect the tracked scope**

Run: `git status --short --branch --untracked-files=no && git diff --check && git diff --cached --check`

Expected: only intended source, tests, workflow, and documentation files; no whitespace errors.

- [ ] **Step 2: Review the release topology and remote refs**

Run: `git ls-remote origin refs/heads/main refs/tags/v2.1.0 refs/tags/desktop-latest`

Expected: `v2.1.0` does not exist before release; `main` and `desktop-latest` baselines are recorded.

- [ ] **Step 3: Resolve all P0-P2 release blockers**

Expected: independent code, documentation, and CI reviewers report no remaining P0-P2 issues.

### Task 4: Commit and Publish Git Refs

**Files:**

### Task 5: CI and Release Readback

**Files:**
- Verify: GitHub Actions and GitHub Releases

**Interfaces:**
- Consumes: pushed `main` and `v2.1.0` refs
- Produces: signed/notarized installers, updater metadata, formal release, and rolling release

- [ ] **Step 1: Track both workflow runs**

Run: `gh run list --workflow build-desktop.yml --limit 10 --json databaseId,headBranch,headSha,status,conclusion,url`

Expected: `main` and `v2.1.0` runs for the release commit both complete successfully.

- [ ] **Step 2: Read back formal release**

Run: `gh release view v2.1.0 --json tagName,url,targetCommitish,assets`

Expected: macOS DMG/ZIP/update metadata for arm64 and x64 plus Windows EXE/blockmap/update metadata are present.

- [ ] **Step 3: Read back rolling release**

Run: `gh release view desktop-latest --json tagName,url,targetCommitish,assets`

Expected: exactly the three manual installers are present and the moving tag resolves to the release commit.
