# AI 生视频工作台：视觉与品牌规范

> 用途：约束后续 HTML 视觉锚点与正式页面的视觉语言。基础来源为 `app/src/renderer/App.vue` 和 `app/src/renderer/views/AiImageWorkbench.vue`，目标是延续抓虾现有工具型桌面产品，而不是创建新的营销视觉体系。

## 1. 视觉定位

- **产品气质**：密集、克制、专业、可长时间使用的桌面生产工具。
- **结构心智**：左侧完成输入与参数配置，右侧观察结果与任务队列；历史从抽屉进入，不形成第三列。
- **品牌识别**：以抓虾现有暗色层级和高识别度橙色作为主要品牌信号。
- **避免方向**：营销落地页、大面积渐变、玻璃拟态、超大留白、装饰性三维元素、用 emoji 充当主要功能图标。

## 2. 设计依据

### 2.1 应用壳

来自 `app/src/renderer/App.vue:469-505`：

- 标题栏高度：`40px`
- 展开侧栏宽度：`168px`
- 收起侧栏宽度：`56px`
- 全局字体：`-apple-system, "PingFang SC", "Microsoft YaHei", sans-serif`
- 全局基准字号：`13px`
- 全局圆角基准：`10px`

### 2.2 AI 生图工作台

来自 `app/src/renderer/views/AiImageWorkbench.vue`：

- 左侧输入、右侧结果的桌面双栏模式。
- AI 生图现状的右侧内部包含主结果区与紧凑历史栏；这只是参考事实，新工作台只继承双栏语法和任务卡视觉，不继承常驻主结果、当前选中任务区或第三列历史栏。
- 新工作台在 `1060px` 以下固定切换为“输入与参数 / 结果与队列”页签模式。
- 视觉密度偏工具型，标题、控件、辅助说明与状态信息层级清晰。

- 状态颜色必须同时配合文字或图标，不依赖颜色作为唯一信息载体。
- 不新增与现有品牌冲突的蓝紫主色。

## 5. 页面骨架

### 5.1 应用壳与视觉锚点边界

- 正式页面必须嵌入现有抓虾应用壳，不在 `AiVideoGenerationWorkbench.vue` 内再次制作全局导航。
- 实际应用中的一级菜单顺序固定为 `AI 生图 / AI 生视频 / AI 视频工作流`，**AI 生视频** active 状态使用橙色文字或图标 + `--accent-soft` 背景。
- 独立 HTML 视觉锚点必须展示完整应用壳侧栏，帮助实现者理解一级入口顺序、内容区起点和工作台的真实可用宽度；页面标题区不再复制一组局部导航按钮。
- 视觉锚点沿用 `40px` 标题栏、`168px` 展开侧栏和 `56px` 收起侧栏的真实尺寸，并演示侧栏收起交互。
- `760px` 以下按现有 AI 工作台规则把一级菜单移动到底部横向导航；这是应用壳响应式行为，不由工作台组件自行实现。

### 5.2 工作台桌面布局

- 页面内边距：`18px 20px`
- 页面主间距：`14px`
- 左侧输入与参数栏：`360-380px`，固定或受限宽度
- 右侧结果与队列栏：最小宽度 `520px`，占据剩余空间
- 整体高度应适配应用内容区，并允许左右区域独立滚动。

建议网格：

```css
.workbench {
  display: grid;
  grid-template-columns: minmax(360px, 380px) minmax(520px, 1fr);
  gap: 14px;
  min-height: 0;
}
```

### 5.3 左侧操作栏

按从上到下的操作顺序组织：

1. Provider 与模型选择。
2. 生成模式选择。
3. Prompt 输入。
4. 参考图上传、排序与移除。
5. 常用生成参数。
6. 可折叠高级设置。
7. 主操作按钮与必要的提交说明。

布局要求：

- 每个字段先标签、后控件，再放一行必要说明；不要把长说明塞进标签。
- Prompt 文本域最小高度沿用 `132px`。
- 参考图使用紧凑网格或横向缩略图队列，明确顺序编号。
- 模型不支持的参数直接隐藏；如果隐藏会让用户误解，则以禁用态配合一句原因说明。
- 主按钮固定使用清晰动词，例如“生成视频”，并在运行中显示状态而非重复提交。

### 5.4 右侧结果与队列

桌面态右栏是一块完整的结果与队列区域，只包含同级平铺任务卡片网格；不拆出永久历史栏，不形成第三列，也不显示常驻大预览、当前选中任务 inspector 或“上方详情 + 下方卡片”的上下分区。

卡片网格规则：

- 宽屏 `>=1181px`：三列。
- 中屏 `1060-1180px`：两列。
- 移动与窄屏 `<1060px`：一列。
- 新创建任务插入网格顶部；任何宽度下都不自动打开详情 modal。

任务卡片至少包含状态、封面或状态占位、模型、Prompt 摘要、阶段说明、创建时间和当前可用操作。整张卡片支持 click、Enter 和 Space 打开同一个居中详情 modal，并提供明确的 `查看详情` 文案。卡片获得键盘焦点时使用橙色 focus ring。

`重试`、`复制为新任务`、`重新归档`、`打开文件`、`日志` 等卡内按钮是独立交互，必须阻止卡片点击事件冒泡；触发这些按钮时不得同时打开详情。空态应解释“从左侧输入 Prompt 或添加参考图开始”，不使用夸张插画占满结果区。

## 6. 面板与控件

### 6.1 面板

- 背景：`#1c1c22`
- 边框：`1px solid #2e2e3a`
- 圆角：`8px`
- 内边距：`14px`
- 面板之间依靠间距与边框区分，不叠加重阴影。

### 6.2 输入控件

- 常规高度：`36px`
- 水平内边距：`10px`
- 圆角：`8px`
- 背景：`#242430`
- 边框：`1px solid #2e2e3a`
- 文字：`#e2e0f0`
- 占位符：`#8e8ca4`
- focus：橙色边框，并增加克制的橙色外圈。

### 6.3 按钮

- 主按钮：橙色背景 `#FF6B2B`，文字 `#17131A`。
- 主按钮 hover：`#cc5522`。
- 次按钮：`#242430` 背景、`#2e2e3a` 边框、主文字色。
- 危险按钮：默认保持暗色，仅在 hover 或确认态提高红色权重。
- 图标与文字间距建议 `6px`；仅图标按钮必须有可读的 `aria-label` 与 tooltip。

### 6.4 图标

- HTML 原型使用内联 SVG，保持统一的 `16px` 或 `18px` 线性图标体系。
- 主操作图标线宽、圆角与现有应用保持一致。
- 不使用 emoji 作为上传、生成、下载、播放、删除、设置等主要操作图标。
- 应用壳侧栏是例外：为了与当前 `App.vue` 保持一一对应，视觉锚点原样使用现有一级菜单 emoji；该例外不扩散到工作台内部控件。

## 7. 交互状态

每个关键控件与任务卡片都应覆盖以下状态：

| 状态 | 视觉要求 |
|---|---|
| Default | 暗色控件、清晰边框、主次文字分层 |
| Hover | 边框或背景轻微提亮，不移动布局 |
| Active / Focus | 橙色边框或选中背景；键盘焦点清晰可见 |
| Queued / Running | `--accent-soft` 背景、橙色状态文字，可配轻量进度动画 |
| Completed / 已完成 | 绿色状态图标/文字，完成卡片保持与其他卡片同级；点击后在 modal 中播放结果 |
| Failed | 红色状态图标/文字，错误说明与重试操作并置 |
| Disabled | 降低对比度但仍可读，同时禁止 hover 误反馈 |

任务运行时：

- 生成按钮进入禁用或运行态，保留“生成中”文字和小型 spinner。
- 结果区显示任务阶段，不用伪造精确百分比；只有 Provider 返回真实进度时才显示百分比。
- 长时间任务要允许用户切换查看结果队列中的其他任务。
- 排队、生成中、下载归档、已完成、待配置、失败、已取消和已过期卡片都可打开详情；未生成视频的状态在播放器区域显示阶段或错误占位。

## 8. 参考图组件

- 上传区使用虚线边框、暗色背景、内联 SVG 上传图标和简短说明。
- 上传后切换为缩略图列表；缩略图应保持原比例裁切预览，并提供编号。
- HappyHorse R2V 的编号直接对应 Prompt 中 `[Image 1]`、`[Image 2]` 的引用顺序。
- 拖拽排序时提供明确插入位置，不依赖缩略图透明度作为唯一反馈。
- 校验错误就地显示在对应图片附近，例如格式、尺寸、比例、数量或大小不符合要求。
- 涉及真人隐私风险时，使用紧凑警示条，不使用阻断式大弹窗代替必要说明。

## 9. 任务详情 modal 与视频预览

- 详情使用视口居中的 modal，不在结果网格上方保留任何常驻预览舞台。
- modal 包含播放器或状态占位、完整 Prompt、模型与公开参数、normalized/provider 状态、归档信息、脱敏错误和当前状态允许的操作。
- 播放器背景可使用接近纯黑但不脱离页面体系的颜色，四周保留 `12-16px` 安全间距。
- 视频保持真实画幅；竖屏视频居中展示，两侧保持暗色，不拉伸填满。
- 播放控制优先使用浏览器原生能力或与应用一致的轻量自定义控制条。
- 结果未下载、尚未生成或远端地址过期时，播放器区域显示明确阶段、失败或恢复说明，而不是空白播放器或伪造视频。
- modal 支持关闭按钮、点击遮罩和 Escape 关闭；关闭后焦点回到触发它的任务卡片、`查看详情` 按钮或历史项。
- 首版 modal 不提供 `上一条 / 下一条`，任务之间的切换回到卡片网格或历史抽屉完成。

## 10. 历史抽屉

- 历史记录由右侧结果与队列区域的 **历史记录** 操作打开抽屉，不占用主网格列宽。
- 抽屉提供状态、Provider 和时间筛选；默认展示最近记录。
- 每条历史项包含缩略图或占位、状态、Provider/模型、创建时间。
- 历史项 hover 与 focus 使用橙色弱背景和清晰边框，不维持跨 modal 的常驻选中态。
- 运行中、`completed / 已完成`、失败状态分别使用橙、绿、红，但同时显示规范化状态文字。
- 历史项高度应紧凑，避免每条卡片复制完整 Prompt；Prompt 可截断为一至两行。
- 关闭抽屉后焦点回到触发按钮；抽屉不改变桌面两栏结构。
- 点击历史项时先关闭抽屉，再打开与任务卡片相同的居中详情 modal。

## 11. 响应式规则

### 11.1 `<1060px`

- 桌面双栏固定切换为“输入与参数 / 结果与队列”两个页签。
- 页面头部纵向堆叠。
- 结果与队列 tab 内保持单列卡片网格；历史仍通过抽屉进入。
- 创建 Job 后自动切换到“结果与队列”，新卡片置顶但不自动打开 modal；任务运行时，该 tab 显示状态点或数量提示。

### 11.2 `760px` 及以下

- 页面内边距调整为 `10px`。
- 主间距调整为 `10px`。
- 操作按钮均分可用宽度。
- 历史抽屉保持覆盖式进入，不占据任务卡片网格宽度。
- 页面标题使用 `18px`。

### 11.3 粗指针设备

在 `@media (pointer: coarse)` 下：

- button、select、input 的最小高度为 `44px`。
- 仅图标按钮的点击区域至少 `44 × 44px`。
- 拖拽排序必须同时提供非拖拽的顺序调整方式。

## 12. 动效与无障碍

- 常规 hover、focus 与展开过渡建议控制在 `120-180ms`。
- 只对 opacity、transform、border-color、background-color 做轻量动画。
- 不让加载动画抢占任务卡片内容或详情 modal 的播放器主体。
- 在 `prefers-reduced-motion: reduce` 下，将动画与过渡压缩到 `0.01ms`，并取消无限位移动画。
- 所有表单标签与控件建立显式关联。
- 键盘顺序遵循从左侧输入到生成，再到右侧结果操作的自然流程。
- 正文、辅助文字与状态文字需保持足够对比度；弱文字不承担关键操作或错误信息。

## 13. 滚动条

延续当前应用壳规则：

- 宽高：`5px`
- thumb：使用边框色 `#2e2e3a`
- 圆角：`3px`
- 左侧输入与参数栏、右侧结果与队列区、历史抽屉只在内容溢出时显示独立滚动，不造成整页横向滚动。

## 14. HTML 视觉锚点验收清单

HTML 视觉锚点至少应让审核者直接看到：

1. 抓虾真实暗色品牌语言与完整应用壳侧栏，明确展示 `AI 生图 / AI 生视频 / AI 视频工作流` 顺序和 **AI 生视频** active 状态。
2. 左侧 `360-380px` 输入与参数栏与右侧纯平铺任务卡片网格的明确双栏关系；没有常驻主预览或当前选中任务区。
3. Provider、模型、模式、Prompt、参考图、参数与生成按钮的完整视觉顺序。
4. 宽屏三列、中屏两列、移动一列的卡片响应式关系。
5. 右侧队列中的排队、生成中、`completed / 已完成`、失败等同级任务卡片，以及从队列打开历史抽屉的入口；不得出现第三列历史栏或上下 master-detail 分区。
6. HappyHorse 与 Seedance 切换时，参数区域存在差异化表达，而非一套表单假装全部通用。
7. 参考图的顺序编号、移除与校验反馈。
8. 工作台操作使用统一控件与可访问标签；应用壳侧栏允许忠实沿用现有 `App.vue` 的 emoji 菜单图标。
9. 整卡 click、Enter、Space 和 `查看详情` 打开同一个居中 modal；卡内独立操作不冒泡。
10. modal 显示播放器/占位、完整 Prompt、参数、状态、归档与操作，并支持关闭按钮、遮罩、Escape 和焦点回收；首版无上一条或下一条。
11. 新任务卡片置顶但不自动打开 modal；历史项关闭抽屉后打开同一 modal。
12. `<1060px` 时“输入与参数 / 结果与队列”页签模式的可说明响应式行为。
13. 页面整体保持密集工具型工作台，不呈现为营销页。
